아무나 깃허브 페이지 만들기

2018-04-05
#github #github_page #git

깃허브페이지를 만들다.

깃허브 페이지를 처음으로 만들었다.
정말 우여곡절 끝에 만들었다.
디자이너 출신에 퍼블을 배우기까지도 무진장 힘들었는데
깃허브 페이지 하나 만드는 것도 그만큼(은 오바지만) 어려웠던거 같다.
막상 만들려고 보니 나는 왜케 오류가 많이 나며..
깃은 뭐고, 노드는 뭐고, 또 깃허브랑 깃허브페이지는 또 뭔차이지??
이걸 과연 다른 웹디, 혹은 일반인(비개발자)가 할 수 있을까? 싶었다.
여러군데 구글링을 해보다 삽질 무진장하고 포기할때쯤 성공하게 되었다.
다른분들은 이런 삽질 없길 간절히 바라며..
웹개발언어 문맹들도 따라할 수 있게끔 친절히 정리해보려고 한다.


들어가기전에..

일단 깃허브페이지를 만들고 hexo를 이용해서 테마를 설치 할 것이다.
그 이유를 주절주절 해보자면.. (넘어가셔도 좋음)

사실 나는 테마를 사용하지않고 블로그를 만들고 싶었는데 결국 테마를 쓰지 않을 수 없었다.
그 이유는 깃허브에서 include가 불가능해서였다. 회사에서 작업하는 환경은 asp기반이라 include가 가능해서 header와 footer, aside에 해당하는 부분들을 include로 관리하고있다.
그래서 당연히 github에서도 가능하겠지? 했는데 무식이 용감인가.. 구글링을 아무리 해봐도 자료는 없고, 결국 노가다로 테스트 여러번 해대가며 파일을 올렸다 내렸다 여러번 반복한 결과 불가능이라는걸 깨달았다.
결국 마크다운으로 작성하는 페이지 테마를 쓰기로하고 수소문한 결과 jekyll과 hexo가 대표적이길래 둘다 해봤다가 그냥 hexo로 가기로 했다. (사실 jekyll로 루비까지 설치하며 해보다가 뭐가 막혔는지 에러를 마구 뱉어내길래 포기함ㅜㅜ)
커스터마이징을 하기 위해서 hexo를 뜯어보며 공부하는 시간도 가졌는데 그 과정은 다음 포스팅에 서술하기로 한다.

아래는 깃허브페이지를 만들기 위해 필요한 프로그램(?)을 우리가 알아야할 부분만 요약해봤다.

  1. git : 간단히 말해 버전관리앱이라고 한다. 더 많은 설명은 구글링을 대신하고 우리는 여기서 git bash를 통해 까만창에 명령어들을 써댈것이다.
  2. git hub : 깃허브는 우리가 접속할 웹계정을 생성할 공간이다. 로컬에서 프로젝트를 작성하고 나의 깃허브웹에 업로드해 누구나 볼 수 있게 한다.
  3. node.js : 자바스트립트 런타임이라고 한다. 간단히 큰 오픈 소스 라이브러리라고 보면 될 것 같다. 일단 노드도 설치해야한다. 이걸 통해 hexo라는 깃허브페이지 테마를 설치 할 것이다.
  4. hexo : 빠르고 간단하고 강력한 블로그 프레임워크라고 사이트에 자기소개하고있다. 지킬기반의 블로그 테마라고 보면 될 것 같다.

포스트 작성하기

포스트를 작성할때도 기본적인 html 상식정도는 알아야 가능한것같다.
나는 1.편집디자이너 -> 2.비개발자(사무직) -> 3.퍼블리셔를 밟아온 사람으로서
앞의 1, 2번 사람의 입장에서 생각 했을 때 이 모든 과정이 너무나도 귀찮을 것 같았다. 깃허브 페이지의 강점은 개발자로서 이것이 포트폴리오가 될수 있다는 점? 아무튼 그것도 개발자일때 얘기니까.. 난 영어섞인 지렁이 글씨가 너무 싫다 하시는 분들은 그냥 편하게 일반 에디터 달려있는 네이버나, 브런치 블로그를 사용하심을 추천한다.

포스트 작성하는 방법도 원래는 마크다운만 있는 줄 알았는데 한 세가지는 되는 것 같다. 이 모든걸 굴리면서도 속도가 그럭저럭 괜찮다는게 신기할따름..

포스트 작성 문법

  1. html 기본문법(마크업)
  2. 마크다운
  3. Hexo Writing(hexo설치 시)

html 기본문법은 퍼블리셔라면 대부분 알터이니 생략하고
마크다운과 hexo-writing사용법은 각 이름에 걸어둔 링크를 참조하도록 한다.
나는 세가지 방법중 마크업과 마크다운을 섞어서 사용하려고 한다.
퍼블리셔로서 blockquate의 디자인과 h1~6의 태그 글씨 크기들이 너무 맘에 안들지만 곧 커스터마이징하는 방법을 포스트하겠다.
